🌲 Agenda 递归

要素agenda(dag, inputs)
Base Caselen(dag)==1: AgentLoop.run()
Recursive StepScheduler → Agent → agenda()
调用者Agent Loop 里的 Agent
参数子 DAG + 子输入
收敛条件depth 递增到 MAX_DEPTH
叶子节点AgentLoop.run()

👁️ 三个视角

Meta Agent 视角

  • 我有一个大任务"写一本书"
  • 我把它拆成 DAG: [大纲, Ch1, Ch2]
  • 我调用 agenda(my_dag, inputs)
  • 等待结果返回

Agent Loop 视角（任意一个 Agent）

  • 我收到了一个 Task"写第一章"
  • 我有一个 Workspace（input/ workspace/ output/）
  • 我可以调用 agenda(sub_dag, inputs) 继续分解
  • 我也可以直接执行，写完输出到 output/
  • 我不知道外面还有多少层 agenda()

DAG Runner 视角

  • 我收到了一个 DAG
  • 我并行启动所有就绪的节点
  • 每个节点 = AgentLoop.run()
  • 我不关心 Agent 内部做了什么
  • 我等待所有节点完成，收集产物

⚡ 核心公式

agenda(dag, inputs, depth=0):
  if len(dag.nodes) == 1:
    return AgentLoop(dag.nodes[0], inputs).run()   // Base Case
  else:
    return Scheduler(dag, inputs, depth).run()   // Recursive Step
    // Scheduler 内部的 Agent 可再次调用 agenda()
